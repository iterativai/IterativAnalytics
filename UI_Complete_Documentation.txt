## 1. DESIGN PHILOSOPHY & PRINCIPLES

### Core Design Values
- **Professional Minimalism**: Clean, focused interfaces that prioritize user tasks
- **African-First Design**: Mobile-optimized for African market connectivity
- **Data-Driven Aesthetics**: Visual hierarchy that supports complex business data
- **Conversion-Focused**: Every element designed to guide users toward key actions

### Design Principles
1. **Clarity Over Complexity**: Simple, intuitive interfaces
2. **Consistency Across Modules**: Unified experience across Ventures, Xchange, and Sourcing
3. **Progressive Disclosure**: Show information when users need it
4. **Accessible by Default**: WCAG 2.1 AA compliance built-in
5. **Performance First**: Lightweight components that load quickly

## 6. ANIMATION GUIDELINES

### Motion Principles
- **Purposeful**: Animations should guide user attention
- **Performant**: Use transform and opacity for smooth animations
- **Respectful**: Honor `prefers-reduced-motion` preferences
- **Subtle**: Micro-interactions enhance, don't distract

### Animation Tokens
```css
/* Duration */
--duration-fast: 0.15s;
--duration-normal: 0.25s;
--duration-slow: 0.4s;
--duration-slower: 0.6s;

/* Easing */
--ease-out: cubic-bezier(0.16, 1, 0.3, 1);
--ease-in: cubic-bezier(0.4, 0, 1, 1);
--ease-in-out: cubic-bezier(0.4, 0, 0.2, 1);
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
```

## 8. ACCESSIBILITY STANDARDS

### WCAG 2.1 AA Compliance
- Minimum contrast ratio: 4.5:1 for normal text
- Minimum contrast ratio: 3:1 for large text (18px+ or 14px+ bold)
- All interactive elements keyboard accessible
- Focus indicators visible and consistent
- Screen reader compatible markup

### Accessibility Utilities
```css
/* Screen reader only text */
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

/* Focus visible styles */
.focus-visible:focus-visible {
  outline: 2px solid hsl(var(--primary));
  outline-offset: 2px;
}

/* Reduced motion support */
@media (prefers-reduced-motion: reduce) {
  *,
  ::before,
  ::after {
    animation-delay: -1ms !important;
    animation-duration: 1ms !important;
    animation-iteration-count: 1 !important;
    background-attachment: initial !important;
    scroll-behavior: auto !important;
    transition-duration: 0s !important;
    transition-delay: 0s !important;
  }
}
```

### ARIA Patterns
```html
<!-- Button with description -->
<button 
  aria-describedby="button-description"
  className="btn-primary"
>
  Submit Application
</button>
<div id="button-description" className="sr-only">
  This will submit your startup application for review
</div>

<!-- Form with validation -->
<div>
  <label htmlFor="email">Email Address</label>
  <input 
    id="email"
    type="email"
    aria-required="true"
    aria-invalid={hasError ? "true" : "false"}
    aria-describedby={hasError ? "email-error" : undefined}
  />
  {hasError && (
    <div id="email-error" role="alert" className="text-red-600">
      Please enter a valid email address
    </div>
  )}
</div>

<!-- Loading state -->
<button 
  disabled={isLoading}
  aria-label={isLoading ? "Loading..." : "Submit"}
>
  {isLoading ? (
    <span aria-hidden="true">Loading...</span>
  ) : (
    "Submit"
  )}
</button>
```

## 9. USER FLOW DOCUMENTATION

### Current User Journey
```
Landing Page → Hero Section → Solutions Overview → Features → Demo → Contact
     ↓            ↓              ↓                ↓       ↓       ↓
Entry Point → Value Prop → Service Details → Benefits → Trial → Conversion
```

### Improved User Journey
```
Landing → Role Selection → Personalized Content → Module Deep Dive → Demo → Signup → Onboarding
    ↓         ↓              ↓                    ↓             ↓        ↓         ↓
Entry → Understanding → Personalization → Specific Value → Trial → Convert → Success
```

### Role-Based Flows

#### Founder Flow
1. **Landing**: Clear value proposition for startup founders
2. **Role Selection**: "I'm a startup founder" selection
3. **Personalized Hero**: Business plan & investor readiness focus
4. **Features**: Ventures module highlighted
5. **Demo**: Ventures-specific demo experience
6. **Signup**: Founder onboarding flow

#### Investor Flow
1. **Landing**: Value proposition for investors/VCs
2. **Role Selection**: "I'm an investor" selection
3. **Personalized Hero**: Deal flow & startup intelligence focus
4. **Features**: Xchange module highlighted
5. **Demo**: Xchange-specific demo experience
6. **Signup**: Investor onboarding flow

#### Partner Flow
1. **Landing**: Value proposition for enterprise partners
2. **Role Selection**: "I'm looking for partnerships" selection
3. **Personalized Hero**: Partnership opportunities focus
4. **Features**: Sourcing module highlighted
5. **Demo**: Sourcing-specific demo experience
6. **Signup**: Partner onboarding flow

## 10. IMPLEMENTATION GUIDELINES

### CSS Architecture
```css
/* Use CSS custom properties for theming */
:root {
  --color-primary: hsl(210, 100%, 50%);
  --spacing-unit: 0.25rem;
  --border-radius: 0.75rem;
}

/* Follow component-scoped styling */
.component-name {
  /* Layout properties */
  display: flex;
  
  /* Box model */
  padding: calc(var(--spacing-unit) * 4);
  
  /* Visual properties */
  background-color: var(--color-primary);
  border-radius: var(--border-radius);
  
  /* Typography */
  font-size: 1rem;
  
  /* Transitions */
  transition: all 0.2s ease-out;
}
```

### Component Guidelines
1. **Use TypeScript interfaces** for all props
2. **Include JSDoc comments** for complex components
3. **Implement proper error boundaries**
4. **Use React.memo()** for performance optimization
5. **Include accessibility props** by default

### File Organization
```
components/
├── layout/
│   ├── Navbar.tsx
│   ├── Footer.tsx
│   └── Container.tsx
├── sections/
│   ├── HeroSection.tsx
│   ├── FeaturesSection.tsx
│   └── CTASection.tsx
├── ui/
│   ├── button.tsx
│   ├── card.tsx
│   └── input.tsx
└── hooks/
    ├── useScrollPosition.ts
    └── useTheme.ts
```

### Performance Guidelines
1. **Lazy load non-critical components**
2. **Optimize images with proper sizing**
3. **Use CSS transforms for animations**
4. **Minimize bundle size with code splitting**
5. **Implement proper caching strategies**

### Testing Guidelines
1. **Unit tests for utility functions**
2. **Component tests with React Testing Library**
3. **Accessibility tests with axe-core**
4. **Visual regression tests**
5. **Performance monitoring**
